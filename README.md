+# 10k Chess — массовая шахматная доска в браузере
+
+Проект реализует «массовые шахматы» в браузере: тысячи игроков одновременно двигают фигуры на одной общей доске. Это не классическая партия 1‑на‑1, а непрерывная мультиплеерная сессия с респавном короля, случайным спавном нейтральных фигур и общим лидербордом. Сервер транслирует изменения состояния доски по WebSocket, а клиент рисует огромное поле на `<canvas>` и обрабатывает пользовательский ввод.
+
+## Ключевые возможности
+
+- **Единая доска на всех**: общее состояние доски хранится на сервере и синхронизируется с каждым клиентом.
+- **До 10 000 игроков (маркетинговая цель)**: архитектура построена на бинарных сообщениях и uWebSockets.js для высокой пропускной способности.
+- **Динамика на общей доске**:
+  - Король каждого игрока респавнится после смерти.
+  - Нейтральные фигуры периодически появляются на пустых клетках.
+  - Захват короля приводит к «нейтрализации» команды на доске и начислению киллов.
+- **Встроенный чат и лидерборд**.
+- **Мобильное управление**: виртуальный джойстик и кнопки масштабирования.
+
+## Технологии
+
+- **Node.js (ES Modules)** — серверная логика.
+- **uWebSockets.js** — WebSocket-сервер с высокой производительностью.
+- **HTML5 Canvas** — рендеринг доски/фигур.
+- **Vanilla JS** — весь фронтенд без фреймворков.
+- **Cloudflare Turnstile** — защита входа (капча) перед подключением.
+
+## Принцип работы
+
+### 1. Серверная модель
+
+- Сервер хранит два массива одинакового размера `board` и `teams`:
+  - `board[x][y]` — тип фигуры (0 — пусто, 1..6 — пешка/конь/слон/ладья/ферзь/король).
+  - `teams[x][y]` — id команды/игрока, владеющего фигурой.
+- Подключившийся игрок получает **полный снапшот** доски, затем — только **дельты** (ходы и изменения клеток).
+- В серверной логике реализованы:
+  - генерация ID игрока,
+  - проверка легальности ходов,
+  - респавн короля,
+  - нейтрализация команды после смерти короля,
+  - антиспам для чата,
+  - ограничение частоты ходов.
+
+### 2. Клиентская модель
+
+- Рендеринг доски в реальном времени на `<canvas>`.
+- Управление камерой: перемещение, масштаб (мышь/клавиши/мобильный джойстик).
+- Обработка кликов и перетаскивания для ходов.
+- Интерполяция движения фигур и визуальная подсветка выбранной фигуры.
+
+### 3. Сетевой протокол (бинарный)
+
+Обмен идёт в виде `Uint16Array` / `Uint8Array`:
+
+- **Полный снапшот доски**: при подключении (длина > 10). Первый элемент — `selfId`.
+- **Ход фигуры**: массив из 5 элементов (10 байт): `[startX, startY, endX, endY]`.
+- **Установка фигуры**: массив из 4 элементов (8 байт): `[x, y, piece, team]`.
+- **Чат-сообщение**: начинается с маркера `47095` + текст.
+- **Лидерборд**: начинается с маркера `48027`.
+- **Нейтрализация команды**: `64535, 12345` + список id.
+
+Протокол минималистичен и оптимизирован для размера сообщений.
+
+## Структура проекта
+
+```
+.
+├── client/             # фронтенд (canvas + UI)
+│   ├── assets/         # изображения фигур и аудио
+│   ├── client.js       # отрисовка, анимации, логика камеры
+│   ├── input.js        # управление, чат, лидерборд
+│   ├── networking.js   # WebSocket-протокол
+│   ├── style.css       # UI и базовые стили
+│   └── index.html      # точка входа
+├── server/             # WebSocket + HTTP сервер
+│   ├── index.js        # логика сервера и протокол
+│   └── badwords.js     # фильтр чата
+├── shared/             # общие константы и правила
+│   └── constants.js    # размеры доски, генерация ходов
+├── package.json
+└── README.md
+```
+
+## Важные файлы и их роль
+
+- **`server/index.js`** — ядро сервера: хранение состояния, обработка подключений, ходов и чата.
+- **`client/client.js`** — рендеринг и визуализация игровой доски.
+- **`client/networking.js`** — обработка входящих сообщений и отправка действий.
+- **`shared/constants.js`** — единая логика правил движения фигур (используется и сервером, и клиентом).
+
+## Как запустить локально
+
+1. Установите зависимости:
+   ```bash
+   npm install
+   ```
+2. Запустите сервер:
+   ```bash
+   npm start
+   ```
+3. Откройте в браузере: `http://localhost:3000`
+
+> Примечание: для production‑окружения нужно настроить ключ капчи Turnstile в `server/index.js`.
+
+## Конфигурация и параметры
+
+В `shared/constants.js`:
+
+- `boardW`, `boardH` — размеры доски (по умолчанию 64×64).
+- `moveCooldown` — минимальный интервал между ходами (1.5с).
+- `respawnTime` — время респавна короля (5с).
+
+В `server/index.js`:
+
+- `captchaSecretKey` — секретный ключ Turnstile.
+- `PORT` — порт запуска (по умолчанию 3000).
+
+## Рекомендации для дальнейшего изучения
+
+1. **Сначала изучить `shared/constants.js`** — здесь базовая логика ходов и размеры доски.
+2. **Далее `server/index.js`** — вся игровая логика и сетевое взаимодействие.
+3. **Затем `client/networking.js`** — увидеть соответствие сообщений и действий клиента.
+4. **И только потом `client/client.js`** — визуализация, интерфейс, управление камерой.
+
+### Что важно знать
+
+- Код ориентирован на **быструю синхронизацию** и **минимальный размер пакетов**.
+- Нет сборщика/бандлера — фронтенд подключается напрямую как статические файлы.
+- Вся логика игры централизована на сервере; клиент лишь отображает состояние.
+- Часть логики (правила ходов) вынесена в `shared/`, чтобы сервер и клиент не расходились.
+
